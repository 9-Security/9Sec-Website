Telemetry Flow

Use this static map as the baseline for what the service should eventually expose into your MDR platform.

01

Kernel Sensor

`edrdrv` captures process, file, registry, object, and minifilter activity. This is the highest-value surface for direct-syscall-resistant telemetry.

02

User Service

`edrsvc` orchestrates collection, caching, serialization, control flow, and cloud transport. It is also the main local control and update risk surface.

03

Offline Analyst Layer

This portal stands in for the future MDR console: scope the case, record what is trustworthy, and route the host into VERIS or other forensic lanes.

04

SaaS Readiness

Only after hardening transport, control-plane auth, and update trust should the telemetry move into a shared multi-tenant MDR service.

Immediate Risk Flags

這些是先前程式碼審查已經確認、而且會直接影響產品化安全邊界的項目。

High · Control Plane

Local RPC trust is too weak

The internal JSON-RPC control surface behaves like a management endpoint but relies on a fixed built-in key model instead of real caller authentication.

High · Transport

TLS verification needs hard proof

The legacy HTTP client path must be refactored to enforce certificate validation and hostname verification before any SaaS-grade telemetry transport is enabled.

High · Supply Chain

Update integrity is outdated

Self-update logic still allows weak integrity behavior and should move to signed packages plus modern hash verification before managed rollout.

Medium · Local IPC

IPC surfaces need ACL tightening

Named events, proxy ports, and related local communication paths should be reduced to least privilege so analysts trust the sensor under real adversary pressure.

Offline Analyst Playbook
  1. Define the host scope first: tenant, hostname, incident source, and whether AxEDR is the primary sensor or a supporting endpoint lane.
  2. Decide the evidence route: AxEDR telemetry, VERIS evidence package, HostWitness live collection, or a combined triage path.
  3. Mark which signals are modernization-sensitive: injection hooks, cloud transport, local control RPC, update path, or Windows 11 compatibility.
  4. Record the minimum service promise: detection-only, triage plus hunting, or full managed containment once network isolation and response controls are ready.
