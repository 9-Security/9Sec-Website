VERIS ENDPOINT TRIAGE

Agentless Windows triage that turns endpoint facts into validated evidence packages, a single full report path, trust scoring, and reproducible IR outputs.

ONE_SHOT_SCANNER NO_RESIDENT_AGENT EVIDENCE_FIRST CENTRAL_ANALYSIS

New Endpoint Scan

Create a dedicated scanner when the endpoint has not been collected yet. Central signs a short-lived token, the launcher writes it locally, and the one-shot scanner will not run without it.

  1. 01 Generate a dedicated scanner request.
  2. 02 Download and run it on the Windows endpoint with the signed token.
  3. 03 Review the Central job and full report.
GENERATE_SCANNER

Analyze Existing Package

Use this path when a scanner zip or evidence package already exists. Central validates the signed package contract before rule processing, correlation, and full report generation.

  1. 01 Select the scanner zip or evidence.json.
  2. 02 Assign a case identifier.
  3. 03 Open the generated report or result JSON.
UPLOAD_EVIDENCE

WHAT IT COLLECTS

Process context, user-mode memory heuristics, persistence, network snapshots, artefact indicators, integrity signals, and scan diagnostics.

WHAT IT DOES NOT DO

No real-time blocking, endpoint isolation, kernel collection, resident monitoring, or full disk forensic imaging.

FULL REPORT

Central reports surface rule matches, evidence trust, gap detection, attack-chain candidates, and investigation pivots in one path.

# BEST_FIT

SOC alert triage when an endpoint needs fast, reproducible evidence packaging.
Temporary investigation where a resident agent is not approved or not practical.
IR handoff where analysts need facts, validation status, and reportable reasoning.
